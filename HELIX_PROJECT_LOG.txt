SESSION 2 — SPEED OPTIMIZATION
================================================================================

[PROMPT] "why the fuck is it so slow? will it become accurate?"

[ACTION] Claude diagnosed speed issue:
  - Llama3 (4.7GB) running on CPU = 20-30s per call
  - TWO LLM calls per command (classify + redact) = doubled latency
  - No output cap = model generates verbose responses for simple 2-line task

[FIX 1] Combined classify + PII redact into SINGLE LLM call
  - New INTENT_PROMPT handles both in one shot
  - Reduced from 2 LLM calls to 1 per command

[FIX 2] Added CLASSIFY_OPTIONS to ollama.chat():
  - num_predict: 60 (caps output at ~60 tokens — 2 lines max)
  - temperature: 0 (deterministic = no sampling overhead)
  - num_ctx: 512 (small context window = faster)

[FIX 3] Switched OLLAMA_MODEL from llama3:latest to qwen3.5:2b
  - qwen3.5:2b = 2.7GB vs 4.7GB for llama3
  - Significantly faster on CPU
  - Accuracy acceptable for intent classification

[FILE MODIFIED] core\sentinel\node.py — full rewrite with speed options
[FILE MODIFIED] config\settings.py — OLLAMA_MODEL = "qwen3.5:2b"

[RESULT] Speed improvement:
  Before: ~25-30 seconds per command
  After:  ~15 seconds per command (measured: 15856ms)
  Still slow due to CPU-only inference — GPU would drop to ~2-3s
