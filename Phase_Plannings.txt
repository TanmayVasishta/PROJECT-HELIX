PHASE 3 PRIORITIES:

1. REGEX FAST-PATH in file_manager.py
   - For obvious commands, skip LLM classification entirely
   - Pattern: if re.match(r"open .+", prompt) → instant route to local
   - Target: ~0ms for pattern-matched commands vs current 15s
   - Patterns needed: open, list, move, copy, delete, show, organize, launch

2. NL ACTION PARSER in file_manager.py
   - Replace the execute() stub with real NL→action dispatcher
   - Parse: "move [files] from [src] to [dst]"
   - Parse: "organize my downloads by file type"
   - Parse: "copy [file] to [folder]"
   - Parse: "find all PDFs in [folder]"
   - Use regex + keyword extraction, NOT another LLM call

3. WORKFLOW PROFILES in a new file: core/profiles/profiles.py
   - "code mode" → open VS Code + Spotify + Chrome (GitHub) + HELIX sidebar
   - "study mode" → open PDFs + organize notes folder + silence notifications
   - "presentation mode" → open PowerPoint + hide desktop + mute mic
   - Profiles stored in ChromaDB, triggered by voice/text command
   - Profile format: {name: str, actions: [list of file_manager commands]}

4. WATCHDOG CONNECTED TO ROUTER
   - start_watcher() called at HELIX startup in main.py
   - Watches: Downloads folder + Desktop
   - On new file: Sentinel classifies it → suggests organization
   - Example: "New file detected: resume_final.pdf — move to Documents/Career?"

5. FEW-SHOT PROMPTING for Sentinel accuracy
   - Add 5-10 examples to INTENT_PROMPT
   - Format: "Example: 'move my photos' → INTENT: local"
   - Improves classification from ~80% to ~95%+

6. LOGGING SYSTEM in logs/ folder
   - Every command + intent + response + timestamp
   - Every PII detection event
   - Every cloud vs local routing decision
   - Format: JSON lines for easy parsing
